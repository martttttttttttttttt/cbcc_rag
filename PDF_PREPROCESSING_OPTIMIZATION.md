# PDF 预处理优化方案 🚀

> 最核心的提速 + 提准环节

## 核心理念

未经处理的 PDF 只是"图片/纯文本堆砌"，必须先把内容**结构化**，才能让检索引擎精准定位信息。

---

## 一、基础处理：文本提取 + 降噪

### 1. 提取纯文本

**优先使用原生文本提取，避开 OCR**（除非是扫描件）
- OCR 易出错且慢
- 原生提取更准确、更快速

**推荐工具库：**

| 语言 | 推荐库 | 备选库 |
|------|--------|--------|
| Python | PyMuPDF (fitz) | pdfplumber |
| Java | Apache PDFBox | - |
| Node.js | pdf-parse | pdf-poppler |

### 2. 文本清洗

**移除无意义字符：**
- 多余换行符
- 多余空格
- 水印文字
- 页眉页脚重复内容

**标准化格式：**
- `§28` → `section 28`
- `SFAT 2021-5` → `SFAT_2021-5`
- `2021/06/02` → `2021-06-02`

**法律术语归一化（同义词映射）：**
- `SFC` ↔ `Securities and Futures Commission`
- `SFAT` ↔ `Securities and Futures Appeal Tribunal`
- 避免"搜全称找不到缩写、搜缩写找不到全称"

## 二、待实现优化

### 1. 智能分块策略
- 按法律文档结构分块（Section、Paragraph）
- 保留段落号（§）作为元数据
- 避免切断完整的法律论述

### 2. 元数据增强
- 提取文档日期、案件编号、当事人
- 识别文档类型（裁决、决定、通知等）
- 标注法律条款引用

### 3. 索引优化
- 建立术语倒排索引
- 支持同义词扩展搜索
- 加权核心法律术语
